# OTTO Recommender System Kaggle 竞赛规则中文参考

原链接：[https://www.kaggle.com/competitions/otto-recommender-system/rules](https://www.kaggle.com/competitions/otto-recommender-system/rules)

说明：本文件是便于项目研究使用的中文参考译文/整理版，不构成法律意见；如与 Kaggle 页面原文不一致，应以原文为准。

## 规则要点

- 竞赛主办方：OTTO / Kaggle 页面所列竞赛实体。
- 参赛者需遵守 Kaggle 通用竞赛规则、页面上的专项规则，以及 Kaggle 服务条款。
- 参赛账号必须是单一、唯一的 Kaggle 账号；不得通过多个账号提交或让代理账号提交。
- 如允许组队，参赛者只能加入一个团队；团队成员必须分别注册并确认团队邀请。
- 团队合并需满足人数上限、提交次数限制、合并截止日期和其他规则要求。
- 竞赛数据可用于竞赛、Kaggle 论坛、学术研究和教育等商业或非商业目的，但仍受页面规则和数据许可证约束。
- 竞赛数据许可证注明为 [CC-BY 4.0](https://creativecommons.org/licenses/by/4.0/)；若许可证条款与竞赛规则冲突，以竞赛规则约束竞赛使用。
- 参赛者应采取合理措施防止未同意规则的人访问竞赛数据，并在发现未经授权传播或访问时通知 Kaggle。
- 可以使用外部数据，但外部数据必须公开、所有参赛者都能免费且平等访问。
- 竞赛期间不得在不同团队之间私下共享与竞赛数据相关的源码或可执行代码，除非完成正式团队合并。
- 可以公开分享竞赛代码，但应在 Kaggle 竞赛相关论坛或 Notebook 中公开给所有参赛者，并且不得侵犯第三方知识产权。
- 使用开源代码时，通常要求代码许可证经 Open Source Initiative 批准，且不限制商业使用。
- 优胜者由私榜排名决定，并以遵守规则为前提；公开榜基于公开测试集，私榜基于私有测试集。
- 若出现并列，较早提交者获胜。
- Kaggle 或主办方可因合规问题移除团队或取消资格。
- 获奖者可能需要提交最终模型软件代码和相关文档，以便复现获奖提交。
- 获奖者需授予主办方专项规则中列明的获奖提交许可，并声明自己有权授予该许可。
- 获奖者需按要求签署领奖、资格、许可、税务等相关文件。
- 税费由获奖者承担；奖项可能按法律要求预扣税款。
- 主办方和 Kaggle 可在法律允许范围内将获奖者姓名和肖像用于宣传。
- Kaggle 和主办方可按隐私政策和竞赛管理需要处理参赛者个人信息。
- 参赛者保证提交作品为原创或拥有相应权利，不侵犯第三方权利，也不违反适用法律。
- 如竞赛因技术故障、作弊、欺诈、未授权干预等原因无法按计划运行，主办方可取消、终止、修改或暂停竞赛。
- 参与竞赛、提交作品或获奖不构成与主办方或 Kaggle 的雇佣合同或聘用邀请。
- 除非专项规则另有规定，与规则相关的争议适用德国法律，并由德国汉堡法院专属管辖。

## 数据与代码条款摘译

### 竞赛数据

“Competition Data” 指竞赛网站提供、用于竞赛目的的数据或数据集，也包括竞赛网站上提供的原型或可执行代码。竞赛数据包含私有和公开测试集，但参赛者不会得知哪些数据属于哪个集合。

参赛者可以为任何商业或非商业目的访问和使用竞赛数据，包括参加竞赛、在 Kaggle 论坛中使用，以及用于学术研究和教育。主办方保留取消任何未按竞赛网站和规则许可使用数据的参赛者资格的权利。

参赛者同意使用合理且适当的措施，防止未正式同意这些规则的人访问竞赛数据。参赛者不得向未参加竞赛的一方传输、复制、发布、再分发或以其他方式提供竞赛数据。

### 外部数据

参赛者可以使用竞赛数据以外的外部数据来开发和测试提交结果。但参赛者应确保这些外部数据是公开可用的，并且所有参赛者都能在竞赛目的下免费、平等地访问。

### 提交代码要求

除非竞赛网站或专项规则另有明确许可，在竞赛期间，参赛者不得私下分享基于竞赛数据或与竞赛相关的源码/可执行代码。该禁止事项包括不同团队之间共享竞赛代码，除非发生正式团队合并。

参赛者可以公开分享竞赛代码，但公开分享不得侵犯任何第三方知识产权。如果选择公开分享代码，应在 Kaggle.com 上与该竞赛专门关联的讨论论坛或 Notebook 中进行，以使所有参赛者受益。

### 获奖者义务

获奖者通常必须向主办方交付用于生成获奖提交的最终模型软件代码及相关文档。交付代码应能够生成获奖提交，并说明成功构建或运行可执行代码所需的资源。若最终模型依赖一般商业可获得的软件，且主办方可无过高成本获取，则可识别这些软件、获取方式和复现所需参数或信息，而不是交付这些商业软件本身。
